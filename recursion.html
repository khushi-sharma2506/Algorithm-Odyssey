Recursion

A function that calls itself is called recursive. Every recursive function has two parts — a base case (when to stop) and a recursive case (call itself with a smaller problem).

Tower of Hanoi

Move n disks from peg A to peg C using peg B as helper. Rules: only move one disk at a time, never place a larger disk on a smaller one.

D&C insight: To move n disks from A to C — move top (n-1) disks to B, move largest to C, move (n-1) disks from B to C.

void hanoi(int n, char from, char to, char aux) { // Base case: only one disk — just move it if (n == 1) { cout << "Move disk 1: " << from << " -> " << to; return; } // Step 1: move top n-1 disks from A to B hanoi(n-1, from, aux, to); // Step 2: move largest disk from A to C cout << "Move disk " << n << ": " << from << " -> " << to; // Step 3: move n-1 disks from B to C hanoi(n-1, aux, to, from); }

Key insight: Each disk added DOUBLES the moves. This is O(2ⁿ) — exponential growth!

Fibonacci — Recursive vs Iterative

fib(n) = fib(n-1) + fib(n-2). Each number is the sum of the previous two. Sequence: 0, 1, 1, 2, 3, 5, 8, 13, 21...

Problem: Recursive Fibonacci recalculates the same subproblems MANY times. fib(5) calls fib(3) twice!

// Recursive — simple but SLOW int fibRecursive(int n) { if (n <= 1) return n; // base case return fibRecursive(n-1) + fibRecursive(n-2); } // Iterative — FAST, O(n) time, O(1) space int fibIterative(int n) { if (n <= 1) return n; int a = 0, b = 1; for (int i = 2; i <= n; i++) { int c = a + b; a = b; b = c; } return b; }
MethodTimeSpaceCalls for fib(10)
RecursiveO(2ⁿ)O(n)177
IterativeO(n)O(1)1 loop

Lesson: Recursion is elegant but can be very inefficient. Dynamic Programming (Ch 6) solves this by storing results.

Factorial

n! = n × (n-1) × (n-2) × ... × 1. Example: 5! = 5 × 4 × 3 × 2 × 1 = 120.

Why it's a perfect recursion example: A problem clearly defined in terms of a smaller version of itself.

int factorial(int n) { // Base case — stop here if (n == 0 || n == 1) return 1; // Recursive case — n! = n * (n-1)! return n * factorial(n - 1); } // Call stack for factorial(4): // factorial(4) -> 4 * factorial(3) // factorial(3) -> 3 * factorial(2) // factorial(2) -> 2 * factorial(1) // factorial(1) -> returns 1 [BASE CASE] // factorial(2) returns 2 // factorial(3) returns 6 // factorial(4) returns 24

Binary Search (Recursive)

Find a target in a sorted array. Look at the middle — if target is smaller, search left half; if larger, search right half. Each call throws away half the array!

int binarySearch(int arr[], int low, int high, int target) { // Base case: search space empty if (low > high) return -1; int mid = (low + high) / 2; if (arr[mid] == target) return mid; // found! if (arr[mid] > target) // target is in left half return binarySearch(arr, low, mid-1, target); return binarySearch(arr, mid+1, high, target); // right half }
CaseTimeWhy
BestO(1)Target is at mid on first check
Average / WorstO(log n)Each call halves the search space
SpaceO(log n)Call stack depth = log n

Key insight: 1000 elements needs only ~10 calls. 1 million elements needs only ~20 calls. That's the power of O(log n)!
