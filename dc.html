Divide & Conquer

Break a big problem into smaller ones, solve each one recursively, then put the results back together. Three steps every time:

DIVIDECONQUERhard
the 3-step pattern (every D&C algo follows this)
DIVIDE
split problem into 2+ smaller pieces
CONQUER
solve each piece recursively
COMBINE
merge sub-solutions into final answer

Merge Sort

DIVIDE: cut array at midpoint into left and right halves.

CONQUER: recursively sort both halves independently.

COMBINE: merge two sorted halves — compare front elements, take smaller one.

void mergeSort(int arr[], int l, int r) { if (l >= r) return; // base: 1 element, already sorted int mid = (l + r) / 2; // DIVIDE mergeSort(arr, l, mid); // CONQUER left mergeSort(arr, mid+1, r); // CONQUER right merge(arr, l, mid, r); // COMBINE }
casetimewhy
bestO(n log n)always splits in half
averageO(n log n)same structure every time
worstO(n log n)input doesn't affect it!
spaceO(n)needs temp array for merging
Quick Sort (D&C view)

DIVIDE: pick a pivot element, move smaller elements left, larger right. Pivot is now in its final position.

CONQUER: recursively sort left side and right side.

COMBINE: nothing! array is already sorted in place.

void quickSort(int arr[], int low, int high) { if (low >= high) return; // base case int pi = partition(arr, low, high); // DIVIDE quickSort(arr, low, pi - 1); // CONQUER left quickSort(arr, pi + 1, high); // CONQUER right // COMBINE: nothing needed! }
casetimetrigger
best / avgO(n log n)pivot splits roughly in half
worstO(n^2)already sorted input, bad pivot
spaceO(log n)only call stack, no extra array
Convex Hull — D&C

Find the smallest convex polygon around all given points. Like stretching a rubber band around pins on a board.

DIVIDE: find leftmost & rightmost point, draw a line. Split remaining points into above-line and below-line groups.

CONQUER: in each group, the point FARTHEST from the line must be on the hull. Split into two new sub-problems around that point.

COMBINE: connect all confirmed hull points in order.

hullRecurse(points, A, B): // find points to the LEFT of line A->B leftSide = [P for P in points if left_of(A, B, P)] if leftSide is empty: B is a hull point // base case return // farthest point MUST be on hull (key insight) P = farthest_from_line(leftSide, A, B) hullRecurse(points, A, P) // CONQUER left hullRecurse(points, P, B) // CONQUER right

Key insight: the farthest point from any line can never be inside the hull — it must be on the boundary.

Kth Smallest / Median Finding

Find the kth smallest element without sorting the whole array. Median = k at position n/2.

DIVIDE: split array into two halves, count elements in left half.

CONQUER: if k fits in left half, search only left. Otherwise search only right. Throw away half the problem each step!

COMBINE: return the found element directly — no merging needed.

methodtimenote
Sort then pickO(n log n)wasteful — sorts everything just to find one element
D&C split-by-half (this file)O(n log n)still O(n log n) because it sorts each half — same cost as sorting!
Quickselect (best D&C)O(n) averageuses partition like Quick Sort — truly faster, O(n²) worst case
Median of MediansO(n) worstguaranteed O(n) but complex to implement
Note on our implementation

The version in dc.cpp sorts each left half at every recursive step, making it O(n log n) — the same as just sorting the array. The real O(n) solution is Quickselect, which uses the partition step from Quick Sort without recursing on both sides.
