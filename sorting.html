Sorting Algorithms

Sorting is the process of arranging elements in a specific order — usually ascending or descending. It is one of the most fundamental problems in computer science.

Bubble Sort

Repeatedly compares adjacent elements and swaps them if they are in the wrong order. The largest element "bubbles up" to the end after each pass.

How it works: Go through the array, compare arr[j] and arr[j+1]. If arr[j] > arr[j+1], swap them. Repeat n-1 times.

void bubbleSort(int arr[], int n) { for (int i = 0; i < n-1; i++) { for (int j = 0; j < n-i-1; j++) { if (arr[j] > arr[j+1]) { swap(arr[j], arr[j+1]); // swap if wrong order } } } }
CaseTimeWhy
BestO(n)Already sorted, no swaps needed
AverageO(n²)Random order, many comparisons
WorstO(n²)Reverse sorted, max swaps
SpaceO(1)No extra array needed

When to use: Only for very small arrays or teaching purposes. Too slow for real use.

Selection Sort

Finds the minimum element in the unsorted part and places it at the beginning. Repeats for remaining elements.

How it works: Find minimum of arr[i..n-1], swap it with arr[i]. Move i forward by 1. Repeat.

void selectionSort(int arr[], int n) { for (int i = 0; i < n-1; i++) { int minIdx = i; for (int j = i+1; j < n; j++) { if (arr[j] < arr[minIdx]) minIdx = j; // track minimum index } swap(arr[i], arr[minIdx]); // place min at position i } }
CaseTimeWhy
BestO(n²)Always scans full unsorted part
AverageO(n²)Same number of comparisons always
WorstO(n²)Same regardless of input
SpaceO(1)In-place, no extra memory

Key insight: Makes very few swaps — only n-1 swaps total. Good when write operations are expensive.

Insertion Sort

Builds the sorted array one element at a time. Like sorting playing cards in your hand — pick one card and insert it in the right place.

How it works: Take element at position i, shift all larger elements one position right, insert it in the correct spot.

void insertionSort(int arr[], int n) { for (int i = 1; i < n; i++) { int key = arr[i]; // element to insert int j = i - 1; while (j >= 0 && arr[j] > key) { arr[j+1] = arr[j]; // shift right j--; } arr[j+1] = key; // insert in correct position } }
CaseTimeWhy
BestO(n)Already sorted, no shifting needed
AverageO(n²)Elements shift on average n/2 positions
WorstO(n²)Reverse sorted, maximum shifts
SpaceO(1)In-place sorting

When to use: Excellent for nearly sorted arrays and small arrays. Used inside other algorithms like TimSort.

Quick Sort

Pick a pivot element, partition the array so all elements smaller than pivot go left and larger go right. Recursively sort both sides.

How it works: Choose pivot (last element), scan from left — move elements ≤ pivot to left side. Place pivot in its final position.

int partition(int arr[], int low, int high) { int pivot = arr[high]; // choose last as pivot int i = low - 1; for (int j = low; j < high; j++) { if (arr[j] <= pivot) { i++; swap(arr[i], arr[j]); } } swap(arr[i+1], arr[high]); return i + 1; // pivot's final position }
CaseTimeWhy
BestO(n log n)Pivot always splits array in half
AverageO(n log n)Good splits on average
WorstO(n²)Already sorted — pivot always smallest/largest
SpaceO(log n)Recursion call stack depth

When to use: Most practical sorting algorithm in real systems. Used in most standard library sort() implementations.

Heap Sort

Uses a binary heap data structure. First builds a max-heap from the array, then repeatedly extracts the maximum element to sort.

How it works: Build max-heap → largest is at root → swap root with last element → reduce heap size → heapify → repeat.

void heapify(int arr[], int n, int i) { int largest = i, l = 2*i+1, r = 2*i+2; if (l < n && arr[l] > arr[largest]) largest = l; if (r < n && arr[r] > arr[largest]) largest = r; if (largest != i) { swap(arr[i], arr[largest]); heapify(arr, n, largest); // fix the affected subtree } }
CaseTimeWhy
BestO(n log n)Always same structure
AverageO(n log n)Heap height is always log n
WorstO(n log n)Guaranteed — input doesn't matter
SpaceO(1)In-place, no extra memory

Key insight: Guaranteed O(n log n) in all cases unlike Quick Sort. But slower in practice due to cache behaviour.

Counting Sort

Not a comparison-based sort! Counts how many times each value appears, then reconstructs the sorted array from the counts.

How it works: Create count array of size (max-min+1). Count each element. Prefix sum the counts. Place elements in output.

void countingSort(int arr[], int n) { int max = *max_element(arr, arr+n); int min = *min_element(arr, arr+n); int range = max - min + 1; vector<int> count(range, 0); for (int i = 0; i < n; i++) count[arr[i]-min]++; // count occurrences int idx = 0; for (int i = 0; i < range; i++) while (count[i]-- > 0) arr[idx++] = i + min; }
CaseTimeWhy
All casesO(n + k)k = range of values
SpaceO(k)Needs count array of size k

When to use: When values are integers in a small known range. Faster than O(n log n) when k is small. Cannot sort floating point numbers.
